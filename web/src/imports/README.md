## 一、文档包说明

本 PRD 文档包以**花小猪打车**真实业务为蓝本，深度融合网约车行业十年产品经验，完整复刻了网约车平台的核心业务链路，并在原生业务基础上增加了 **AI 智能能力**（智能客服、智能调度、智能路线规划、MLOps 治理）。

### 1.1 编写原则

- **真实还原**：基于花小猪实际业务模式（一口价、 young 用户定位、轻运营策略）进行复刻。
- **功能合理**：所有功能设计遵循网约车行业通用逻辑与商业合理性。
- **数据驱动**：每个需求项均包含可量化的验收标准（SLO/KPI）。
- **工程可落地**：数据字典达到 L3 级别（可直接指导数据库设计）。
- **AI 增强**：在原有业务基础上，系统化引入 AI 能力，提升运营效率与用户体验。

### 1.2 文档结构

| 序号 | 文档 | 覆盖模块 | 需求项数 |
|---|---|---|---|
| 01 | `01-passenger-app.md` | 乘客端 | 6 项（登录注册、地址管理、叫车下单、订单跟踪、支付发票、评价申诉） |
| 02 | `02-driver-app.md` | 司机端 | 6 项（登录出车、听单大厅、接单导航、行程执行、收入账单、客服上报） |
| 03 | `03-operations-console.md` | 运营后台 | 6 项（组织权限、订单看板、司机车辆、营销活动、客诉质量、规则配置） |
| 04 | `04-ai-capabilities.md` | AI 中台 | 4 项（智能客服、智能调度、智能路线规划、MLOps 治理） |

### 2.2 核心业务链路

```
乘客下单 → 预估计价 → 订单创建 → 智能调度（AI派单） → 司机接单
    ↓                                                      ↓
支付完成 ← 行程结束 ← 轨迹上报 ← 导航接驾 ← 司机到达 ← 开始行程
    ↓
评价/发票/客服
```


### 3.3 术语表

| 术语 | 说明 |
|---|---|
| 一口价 | 下单时锁定价格，行程中不因堵车/绕路变化（除修改终点外） |
| 派单 | 系统将订单分配给司机的过程 |
| 接驾 | 司机接单后前往乘客上车点的过程 |
| 行程 | 乘客上车后至到达终点的过程 |
| 服务分 | 司机服务质量综合评分（满分 100） |
| 热力图 | 按网格展示区域订单密度的可视化工具 |
| 强派 | 平台强制指派订单，司机不可拒绝 |
| 隐私号 | 司机与乘客通话的中转虚拟号码 |
| PSI | Population Stability Index，群体稳定性指数，用于监控特征漂移 |
| MAPE | Mean Absolute Percentage Error，平均绝对百分比误差 |
